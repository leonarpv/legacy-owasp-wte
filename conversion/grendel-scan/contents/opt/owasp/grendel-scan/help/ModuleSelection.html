Test Module Selection

This tab is pretty self-explanatory. There's no problem with enabling most of the modules, but some require special consideration. Each module has useful documentation, but some cautions are summarized below:

Spider: The spider can cause problems for poorly designed websites. For example, if there is a link that reads, "Click here to delete the database", the spider will still follow the link. The HTTP standard states that requests that change data should only be sent over POSTs, but that is not always followed. If that advice is followed by the website, then the "HTML tag requester" should be fine. The "Form baseline" could easily cause problems though. The "URL-regex" module won't cause any problems over the tag requester, but is a little slow, and usually isn't needed.

File enumeration: The directory enumerator, and especially the file enumerator can be very time consuming. The backup file enumerator is faster, and can have more useful results.

Information leakage: No problems.

Session management: No problems.

XSS: The exhaustive nature of the "XSS - Query" module can take a noticeable amount of time in a complicated application.

SQL Injection: If a SQL tautology is tested against a query that does something dangerous (say deleting a record), then it could cause a problem if the query is vulnerable. This can be mitigated using URL blacklists and forbidden query parameters.

Nikto: Just as safe as the real Nikto.

Miscellaneous attacks: No problems

Application architecture: Keep in mind that the mirroring module will require as much space as the website is large.

Web Server Configuration: No problems
